## §5 콘텐츠 데이터 계약 v2 (R1 + R4)

**시스템 고정 (멤버 수정 불가 — §6 공용 영역)**:
1. 위치·파일명 = `content/<종류>/YYYY-MM-DD-<작성자id>-<슬러그>.md` (종류 enum = **{기사, 세미나}** — 실습 폐지)
2. frontmatter — 공통 = `{title, author, date}`
   - 기사 += `{source_url, source_name, 성격 ∈ {뉴스·동향, 심층 분석, 활용법·튜토리얼, 도구·프롬프트} (단일 필수), 주제 ∈ {에이전트, 모델·플랫폼, 워크플로·자동화, 거버넌스·리스크, 시장·생태계} (단일 필수 — 2026-07-23 3차: 구 영역[경영기능 7값]은 실소재와 불일치로 교체, 적용 사례 축적 시 경영 축 재도입 검토), 지금써먹기(boolean 선택), 이미지(문자열 선택 — 썸네일), 고정(boolean 선택 — 허브 뷰 상단 핀)}` — 구 tags·용도/기술 폐지(기존 파일 이관). 성격 4값 정의·판정 질문 = `CONTRIBUTING.md`(기고 가이드, repo 루트)
   - 세미나 += `{회차(number), 유형 ∈ {인지, 실습}, 발원기사(선택), 슬라이드(선택 URL), 요점(선택 배열 1~4 — 아코디언 요약), 장소(선택)}`. 예정/과거 판정 = `date ≥ 오늘`(자동 — 상태 필드 없음)
3. 세미나 본문 규칙: `유형 = 실습 → 필수 헤딩 {## 준비, ## 진행, ## 재현 가이드} 전부 존재` (CI 검증). 인지 = 자유 구성
4. 스키마 유일 원천 = `app/src/content/schema.js`(검증기·앱 로더 공용)

**멤버 재량 (판정 없음)**: 생산 방식 · 본문 구성(§5-3 제외) · 기고 빈도·소재.

**검증**: `필수 필드 결측 ∨ enum 밖 값 ∨ 파일명 패턴 불일치 ∨ §5-3 위반 → CI 실패`
