## §1 의의 [비규범]

사이트 = 광운대 경영학부 MIS×AI 스터디의 허브이자 실습장. 세미나(인지·실습 교차)의 산출물이 쌓이고, 스터디원이 각자의 AI 활용 방식을 개발해 직접 적용하는 무대. 정보 전달을 넘어 "스터디원 = 생산 주체"를 저장소 구조로 강제한다.

## §5 콘텐츠 데이터 계약 (R1 + R4)

**시스템 고정 3 (멤버 수정 불가 — §6 공용 영역)**:
1. 위치·파일명 = `content/<종류>/YYYY-MM-DD-<작성자id>-<슬러그>.md` (종류 enum = {기사, 세미나, 실습})
2. frontmatter 스키마 — 공통 = `{title, author, date}` / 기사 += `{source_url, source_name, tags ⊂ {에이전트, 채용시장, 거버넌스, 도구, 기타}}` / 세미나 += `{회차(number), 유형 ∈ {인지, 실습}}` / 실습 += `{연계회차(number), tools[]}`
3. 목록·상세 렌더 코드 = 시스템 소유

**멤버 재량 3 (규범 판정 없음)**:
1. 생산 방식 — 수동 작성 / 본인 프롬프트 루틴 / 자동 루프. 자동화의 실행 환경·git 계정 = 멤버 본인 것(시스템 측 수집 인프라 없음)
2. 본문 마크다운 구성
3. 기고 빈도·소재 선택

**검증 규칙**: `필수 필드 결측 ∨ tags·유형이 enum 밖 ∨ 파일명 패턴 불일치 → CI 실패`

## §6 반영 통제 (R4)

- `변경 경로 ⊆ members/<본인id>/ ∪ content/ → 직접 push 허용`
- `변경 경로에 그 외(공용 코드·설정·data/) 포함 → PR + owner 승인` (CODEOWNERS로 강제)
- `CI 실패 → merge·배포 차단`

## §7 개인 페이지 규격 (R1)

- `members/<id>/index.html` 존재 → 빌드 엔트리 자동 등록(글롭 스캔). 폴더 내부 파일 구성 = 멤버 재량.
- 시작 템플릿 = React(자기소개 + 작업물 목록 컴포넌트, 스타일은 로컬 사본으로 동봉). 템플릿 채택 여부 = 멤버 재량(순수 HTML 허용).
- `members/ 내 파일이 공용 src/ 를 import → CI 경고 + PR 리뷰 반려` (공용 변경이 개인 페이지를 깨뜨리는 결합 차단)
- `profile.json` 스키마 = `{id, name, tracks[], intro(1줄), page(bool)}`
