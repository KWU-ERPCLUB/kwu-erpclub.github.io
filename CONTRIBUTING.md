# 기고 가이드 — AI 인사이트

광운대학교 ERP연구회 MIS×AI 스터디 허브. 인사이트(`/insights/`) 기고 규칙·절차.
frontmatter 규칙의 유일 원천 = `app/src/content/schema.js`. 이 문서는 그 사람용 설명.

## 기고 흐름

1. **collaborator 등록** — 오너에게 GitHub 계정 전달, 저장소 쓰기 권한 획득
2. **clone** — `git clone https://github.com/KWU-ERPCLUB/kwu-erpclub.github.io.git`
3. **템플릿 복사** — `app/content/기사/_template.md` → 같은 폴더에 새 파일로 저장
4. **파일명 규칙** — `YYYY-MM-DD-<작성자id>-<슬러그>.md` (예: `2026-08-01-hong-agent-workflow.md`)
   - `<작성자id>` = frontmatter `author` 값과 동일
   - `_` 시작 파일 = 게재·검증 제외(초안·템플릿 보관용)
5. **frontmatter 채움** — 아래 필드 표 참고, 주석 줄 삭제
6. **push** — `content/기사/` 경로는 직접 push 허용(공용 영역 변경은 PR + 오너 승인)
7. **CI 검증** — validate + test + build 게이트. 규칙 위반 = 배포 차단
8. **자동 게재** — main 병합 시 GitHub Pages 자동 배포

## frontmatter 필드

| 필드 | 필수 | 값 |
|------|------|-----|
| `title` | 필수 | 글 제목 한 줄 (개조식·AI투 금지) |
| `author` | 필수 | 작성자id (파일명 프리픽스와 동일) |
| `date` | 필수 | `YYYY-MM-DD` |
| `source_url` | 필수 | 원문 URL |
| `source_name` | 필수 | 출처 이름 (예: Stanford HAI) |
| `성격` | 필수 | 단일값 — 아래 성격 4값 중 1개 |
| `주제` | 필수 | 단일값 — 아래 주제 5값 중 1개 |
| `지금써먹기` | 선택 | boolean (기본 false). true = 지금 바로 적용 가능한 실전형 |
| `이미지` | 선택 | 문자열 — 목록·허브 썸네일 (경로 `/img/기사/x.png` 또는 https URL) |
| `고정` | 선택 | boolean (기본 false). true = 허브 뷰 해당 성격 섹션 최상단 고정 |

## 성격 4값 (단일 선택)

| 성격 | 정의 | 판정 질문 |
|------|------|-----------|
| 뉴스·동향 | 새 소식·출시·시장 변화의 사실 전달 | 최근 새로 생긴 일인가? |
| 심층 분석 | 한 주제를 파고든 해석·시사점 | 해석이 절반 이상인가? |
| 활용법·튜토리얼 | 따라 할 수 있는 절차·실습 | 따라 하면 재현되는가? |
| 도구·프롬프트 | 특정 도구·프롬프트가 주인공인 소개·사용기 | 도구가 주인공인가? |

- 판정 = 질문 하나에 "예"인 성격 선택
- 둘 이상 걸치면 = 글의 무게중심(전달 vs 해석 vs 절차 vs 도구) 기준 1개

## 주제 5값 (단일 선택 — AI 주제 축, 2026-07-23 교체)

- 에이전트 — 에이전트·자동 작업 수행 관련
- 모델·플랫폼 — 모델 출시·업데이트·플랫폼 소식
- 워크플로·자동화 — 하네스·파이프라인·업무 자동화 방법
- 거버넌스·리스크 — 정책·보안·윤리·리스크
- 시장·생태계 — 시장 동향·채용·생태계 변화

## 이미지·고정 필드

- **이미지** = 카드 우측 썸네일(96×72). 생략 시 = 성격 색 타일 + 성격명 자동 표시
- **고정** = `true`면 허브 뷰(전체 탭)에서 해당 성격 섹션 최상단 핀. 상시 노출용 기준 글에만 사용

## 본문 자유 원칙

- 본문 구성 = 기고자 재량 (텍스트·이미지·링크·웹 임베드)
- 이미지 = `![대체텍스트](URL)`, 본문 폭(720px) 자동 축소
- 링크 = `[표시 문구](https://...)`
- 강조·목록·인용(콜아웃) 등 마크다운 문법 사용 가능
- 카피 톤 = 개조식(명사형·구 단위 종결). 과장·감탄·권유형 수사 금지
